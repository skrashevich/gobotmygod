## Новые функции

- **Прокси загрузки файлов** — [API-прокси](/docs/ru/features/api-proxy) теперь включает эндпоинт для загрузки файлов по адресу `/tgapi/file/bot{TOKEN}/{file_path}`. Вы можете использовать его как `base_file_url` в вашей библиотеке для ботов. Файлы стикеров WebP автоматически конвертируются в PNG для совместимости с браузерами.

- **Миграция ботов без настройки** — Направьте бэкенд вашего бота на API-прокси BotMux и вызовите `setWebhook` как обычно. BotMux автоматически зарегистрирует бота, включит режим прокси и начнет пересылать обновления — без ручной настройки. Вызов `deleteWebhook` через прокси отменяет процесс. Подробнее в разделе [обратный прокси](/docs/ru/features/reverse-proxy).

## Улучшения

- **Безопасная обработка методов API** — API-прокси теперь перехватывает методы `logOut` и `close` вместо их пересылки в Telegram. Это предотвращает случайное отключение токена (блокировка на 10 минут) или завершение работы экземпляра бота при вызове этих методов через прокси. См. [перехватываемые методы](/docs/ru/features/api-proxy#intercepted-methods).

- **Перехват getUpdates для длинного опроса** — Когда длинный опрос включен для бота, вызов `getUpdates` через API-прокси теперь возвращает обновления из внутренней очереди BotMux вместо прямого опроса Telegram. Это предотвращает конфликты между прокси и циклом опроса вашего бота.
