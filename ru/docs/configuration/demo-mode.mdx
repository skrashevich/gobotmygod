---

Демо-режим создаёт изолированные окружения для публичных демонстраций.

## Включение

```bash
# Через флаг
./botmux -demo

# Через переменную окружения
DEMO_MODE=true ./botmux

# Docker
docker run -e DEMO_MODE=true -p 8080:8080 botmux
```

## Поведение

- Используется отдельная база данных `demo.db`
- Вход: `demo` / `demo` (смена пароля не требуется)
- 3 предварительно настроенных демо-бота
- Используется фиктивный Telegram API (`telegram-bot-api.exe.xyz`)
- Сессии истекают после 30 минут неактивности
- `/api/health` возвращает `{"mode": "demo"}`, чтобы фронтенд автоматически подставлял учётные данные
