## Новые функции

- **Маршрутизация сообщений на основе LLM** — Теперь вы можете маршрутизировать сообщения между ботами с помощью LLM, совместимой с OpenAI. Настройте провайдера LLM и описания ботов, и BotMux будет интеллектуально решать, какой бот должен обработать каждое входящее сообщение.

- **Мосты протоколов с поддержкой Slack** — Свяжите ваших Telegram-ботов с внешними платформами обмена сообщениями. Первая поддерживаемая интеграция — Slack, позволяющая пересылать сообщения между чатами Telegram и каналами Slack.

- **Демо-режим** — Попробуйте BotMux без полной установки. Демо-режим создает изолированные эфемерные сессии с предзаполненными учетными данными, чтобы вы могли мгновенно изучить панель управления и функции.

- **Потоковая передача сообщений в реальном времени** — Сообщения теперь передаются на панель управления в реальном времени через Server-Sent Events. Вам больше не нужно вручную обновлять страницу, чтобы увидеть новые сообщения — они появляются автоматически по мере поступления.

- **Просмотр логов приложения** — Администраторы теперь могут просматривать и получать логи приложения в потоковом режиме прямо в веб-интерфейсе. Просмотрщик логов включает автопрокрутку и доступен из панели администратора.

- **Фильтрация исходного чата для маршрутов** — Правила маршрутизации теперь поддерживают фильтр исходного чата. Вы можете ограничить маршрут так, чтобы он пересылал сообщения только из определенного чата, что дает вам более тонкий контроль над маршрутизацией сообщений между ботами.

- **Аутентификация и управление пользователями** — BotMux теперь включает встроенную систему аутентификации с экраном входа, управлением пользователями и доступом на основе ролей.

- **Управление API-ключами** — Генерируйте и управляйте API-ключами для программного доступа к REST API BotMux.

- **Отслеживание сообщений по ботам** — Сообщения теперь отслеживаются по каждому боту, поэтому вы можете просматривать и искать историю сообщений для каждого бота отдельно.

- **Пользовательский базовый URL Telegram API** — Вы можете переопределить стандартную конечную точку Telegram API на пользовательский URL с помощью флага CLI или переменной окружения. Полезно, если вы используете локальный сервер Telegram Bot API.

- **Поддержка Docker** — BotMux поставляется с Dockerfile, docker-compose.yml и мультиплатформенными сборками Docker.

- **Ответы на сообщения и лайтбокс для медиа** — Теперь вы можете отвечать на сообщения прямо из панели управления. Медиафайлы открываются в лайтбоксе для удобного просмотра с улучшенным рендерингом стикеров и конвертацией WebP в PNG.

## Улучшения

- **Поддержка длинного опроса** — Боты теперь могут использовать длинный опрос для получения обновлений как альтернативу вебхукам. Настройте его для каждого бота через панель управления.

- **Улучшенное определение методов Telegram API** — API-прокси теперь определяет метод Telegram из параметров запроса или полей тела запроса, что повышает надежность при перехвате исходящих сообщений бота.

- **Улучшенные русские переводы** — Обновлены и стандартизированы все строки пользовательского интерфейса на русском языке для единообразия и удобочитаемости.

- **Улучшенная валидация токена бота** — Форма регистрации бота теперь проверяет токены перед сохранением и показывает подсказки по режимам, чтобы помочь вам выбрать между режимами прокси и управления.

## Исправления ошибок

- **Исправлены временные метки сообщений** — Решена проблема, при которой временные метки сообщений отображались некорректно из-за несоответствия секунд и миллисекунд.

- **Исправлена проверка работоспособности Docker** — Проверка работоспособности контейнера теперь использует публичную конечную точку `/api/health` вместо маршрута, защищенного аутентификацией, поэтому состояние работоспособности Docker отображается корректно.

- **Исправлены прокрутка чата и расположение ответов** — Представление чата теперь сохраняет позицию прокрутки при автообновлении, показывает превью сообщений в списке чатов и корректно отображает цепочки ответов.

- **Исправлена обработка типов медиаконтента** — Медиа-прокси теперь корректно определяет типы контента на основе расширения файла с правильными заголовками кэширования.
