## 新功能

- **基于 LLM 的消息路由** — 您现在可以使用兼容 OpenAI 的 LLM 在机器人之间路由消息。配置您的 LLM 提供商和机器人描述，BotMux 将智能决定由哪个机器人处理每条传入消息。

- **支持 Slack 的协议桥接** — 将您的 Telegram 机器人桥接到外部消息平台。首个支持的集成是 Slack，允许您在 Telegram 聊天和 Slack 频道之间转发消息。

- **演示模式** — 无需完整安装即可试用 BotMux。演示模式创建隔离的临时会话并预填凭据，让您可以即时探索仪表板和功能。

- **实时消息流式传输** — 消息现在通过 Server-Sent Events 实时传输到仪表板。您不再需要手动刷新即可看到新消息——它们会在到达时自动显示。

- **应用日志查看器** — 管理员现在可以直接在 Web 界面中查看和流式传输应用日志。日志查看器包含自动滚动功能，可从管理员仪表板访问。

- **路由的源聊天过滤** — 路由规则现在支持源聊天过滤器。您可以限制路由仅转发来自特定聊天的消息，从而更精细地控制机器人之间的消息路由。

- **身份验证和用户管理** — BotMux 现在包含内置的身份验证系统，提供登录界面、用户管理和基于角色的访问控制。

- **API 密钥管理** — 生成和管理 API 密钥，以编程方式访问 BotMux REST API。

- **多机器人消息跟踪** — 消息现在按机器人跟踪，因此您可以独立查看和搜索每个机器人的消息历史。

- **自定义 Telegram API 基础 URL** — 您可以使用 CLI 标志或环境变量将默认的 Telegram API 端点替换为自定义 URL。如果您运行本地 Telegram Bot API 服务器，这将非常有用。

- **Docker 支持** — BotMux 附带 Dockerfile、docker-compose.yml 和多平台 Docker 构建。

- **消息回复和媒体灯箱** — 您现在可以直接从仪表板回复消息。媒体文件在灯箱中打开以便于查看，并改进了贴纸渲染和 WebP 到 PNG 的转换。

## 改进

- **长轮询支持** — 机器人现在可以使用长轮询作为 Webhook 的替代方案来接收更新。可以从仪表板为每个机器人进行配置。

- **改进的 Telegram API 方法检测** — API 代理现在可以从查询参数或请求体字段推断 Telegram 方法，提高了捕获机器人外发消息的可靠性。

- **改进的俄语翻译** — 更新并标准化了所有俄语界面字符串，以保持一致性和可读性。

- **增强的机器人令牌验证** — 机器人注册表单现在会在保存前验证令牌，并显示模式提示以帮助您在代理模式和管理模式之间进行选择。

## 错误修复

- **修复了消息时间戳** — 解决了由于秒与毫秒不匹配而导致消息时间戳显示不正确的问题。

- **修复了 Docker 健康检查** — 容器健康检查现在使用公共的 `/api/health` 端点而不是受身份验证保护的路由，因此 Docker 健康状态可以正确报告。

- **修复了聊天滚动和回复布局** — 聊天视图现在在自动刷新时保留滚动位置，在聊天列表中显示消息预览，并正确渲染回复线程。

- **修复了媒体内容类型处理** — 媒体代理现在可以根据文件扩展名正确确定内容类型，并使用正确的缓存头。
