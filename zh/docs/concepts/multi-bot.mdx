---
title: "多机器人管理"
description: "在单个 BotMux 实例中管理多个 Telegram 机器人"
---

BotMux 在单个实例中管理多个机器人。所有机器人功能完全相同——通过 CLI 添加和通过 Web UI 添加的机器人没有任何区别。

## 添加机器人

### 通过 CLI

```bash
./botmux -token "123456:ABC-DEF..."
```

令牌会自动注册为机器人，并可通过 Web UI 进行配置。

### 通过 Web UI

1. 点击侧边栏中的 **+ ADD**
2. 输入机器人令牌并点击 **CHECK** 验证
3. 为其命名
4. 启用 **Management** 和/或 **Proxy** 模式
5. 如果启用了代理，输入后端 webhook URL
6. 点击 **SAVE**

机器人会立即开始轮询——无需重启。

## 机器人模式

每个机器人可以运行在一种或两种模式下：

| 模式 | 说明 |
|------|-------------|
| **Management** | 聊天跟踪、消息监控、管理操作、数据分析 |
| **Proxy** | 反向代理——将更新转发到后端 URL |

## 机器人解析

API 端点接受 `bot_id` 参数。服务器先检查已注册的机器人，然后回退到 ProxyManager 管理的机器人。

## 聊天隔离

聊天使用复合主键 `(bot_id, chat_id)`。每个机器人拥有独立的聊天视图，即使多个机器人在同一个群组中也是如此。
